Website Style guide
Reference source for theme UI components
Introduction

Use this space to write a brief overview of the purpose and scope of the style guide. Try to communicate the benefits of following it and provide context for the design decisions that have been made.

Also, identify the intended audience, such as designers, developers, and content creators. Explain how to use the guide, including who to contact with questions.

Additionally, you might mention any important technical considerations, such as accessibility, responsive design, and browser compatibility as well.

Logo

The logo section of the style guide should feature high-resolution images of the logo in all its variations - horizontal, vertical, and monochrome. Essential details such as the precise colors used in the logo specified using HEX and/or RGB values, the typeface and any associated typography should also be included.

Finally, be sure to outline clear and concise guidelines on the proper usage of the logo, taking into account factors such as minimum size, appropriate clear space, and suggested background colors.

Color Palette

A color palette should define the complete spectrum of colors that a brand uses to establish its identity. For most use cases, this spectrum can be divided into four categories: (1) primary colors, (2) secondary colors, (3) neutrals, and (4) semantic colors. To ensure accurate reproduction, each color should be represented by its hexadecimal or RGB value.

Additionally, it is crucial to establish a naming convention for the color palette to provide a clear and consistent method of referring to the colors.

Typeset

Specify all typefaces used in the design, including the font family, weight, and size. It is recommended to use no more than two or three typefaces in total. For the majority of text, such as body copy, set one typeface, and the other one(s) for headlines, callouts, quotes, captions and similar elements.

Besides type scale (font, weight, size) good typography should also establish other typographic constructs like fallback options, horizontal and vertical rhythm, text color, responsiveness. A systematic approach here will help define even more design aspects such as icon sizings, margin, padding and spacing values, and general sizing of UI components.

Breakpoints

Based on the content of the website determine the breakpoints starting from the smallest screen you want to support to largest. This is the most important part of responsive design, so it's wise to use website analytics to discern the most commonly used devices from which website is accessed.

Grid System & Layout

The grid system should establish several key elements, including the size and shape of the container, the spacing between columns (known as the gutter space), the number of columns in the layout, the breakpoints at which the layout changes based on the size of the screen, and the margin around the edges of the container.

Closely related to grid system, layout system should specify the various ways in which content is laid out on the page, including options such as a one-column, two-column, or three-column layout, as well as variations such as a layout with a left or right sidebar.

Buttons

Be sure to include all types of buttons (primary, secondary, ghost, cta ...) with all possible states: focused, invalid, disabled, loading, etc.

It's also a good idea to provide guidelines for the placement of buttons, including the distance from other elements and the use of white space.

Forms

Style guide rules for forms should include everything from the form layout (form structure) and choosing appropriate form controls to spacing between elements. They should also take into consideration positioning of labels, different states indicators (focused, disabled, invalid) error handling and validation, etc.

Tables

Table design guidelines should encompass all aspects of table design, from headers and footers (and table captions) to cell shading. It is important to provide clear instructions on text formatting and alignment within cells and whether or not borders should be used.

Depending on expected table sizes more additional elements should also be included such as pagination, table actions, horizontal/vertical scrolling, fixed columns or rows and so on.

Imagery & iconography

A comprehensive set of imagery design guidelines should include all the rules for how you use images, such as image formats, size, resolution and their placement on the page. It's also advisable to provide guidance on where to source images and how to obtain the necessary permissions for using them.

A well-crafted set of iconography guidelines should define the look and feel of icons, rules for creating and using icons in different contexts, and finally a library of approved icons. To keep up with the latest trends and best UX practices try to establish your icon set as svg files instead of using icon fonts.

Misc.

This section serves as a catch-all for any elements that did not fit into the previous sections, such as loaders, pagination controls, breadcrumbs, notification messages, avatars, and others.
